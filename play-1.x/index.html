Servlet API Neden Kullanlmyor?

  • Play Framework ierisinde Servlet API kullanlmamasnn iki ana nedeni var.
    • Tamamen stateless mimariye sahip olmas
    • Kod deiikliklerini otomatik olarak yklemesi
  • Eer uygulama tamamen stateless deilse kod deiikliinden sonra bellek durumunu yeniden oluturmann bir yolu yok.
  • lk srmler HTTP Session ve Servlet Forwarding gibi mekanizmalar olmakszn Servlet API kullanyor ve bir Servlet Container iinde alyordu.
  • Servlet tabanl bileenlerin stateless mimariyi krma olasl ok fazla.
  • Servlet API eski moda "her istek iin bir thread" yapsn kullanyor. Gnmzn bolca Ajax ve long polling arlar ieren uygulamalar iin bu yeterli deil.

Asenkron ler

  • WebSocket 
public class MyWebSocket extends WebSocketController {
  public static void echo() {
    while(inbound.isOpen()) {
      WebSocketEvent e = await(inbound.nextEvent());
      if(e instanceof WebSocketFrame) {
        WebSocketFrame frame = (WebSocketFrame)e;
        if(!e.isBinary) {
          if(frame.textData.equals("quit")) {
            outbound.send("Bye!");
            disconnect();
          } else {
            outbound.send("Echo: %s", frame.textData);
          }
        }
      }
      if(e instanceof WebSocketClose) {
        Logger.info("Socket closed!");
      }
    }
  }
}
